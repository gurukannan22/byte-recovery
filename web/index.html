How It Works

File carving explained — no file system required.

01

Open Raw Disk

Bypasses the file system (NTFS, APFS, ext4) and opens the physical storage device using OS-level APIs — CreateFileW on Windows, direct file access on Unix.

02

Chunk Scanning

Reads the disk in 512 KB sector-aligned chunks. Deleted data remains on disk until overwritten, allowing recovery even after standard deletion or formatting.

03

Signature Matching

Each chunk is scanned for known magic number headers — e.g. FF D8 FF for JPEG, %PDF for PDF — across 30+ file types.

04

File Extraction

When a header is found, bytes are read until a matching footer or size limit is reached. Files are saved organized by type into your output directory.
