How It Works

No file system? No problem.

File carving works at the binary level — recovering data even when the directory structure is gone.

01

Open the Raw Disk

Uses OS-level APIs (CreateFileW on Windows, direct device open on Unix) to read raw physical sectors, bypassing NTFS, APFS, and other file systems entirely.

02

Scan in 512 KB Chunks

Reads the disk in sector-aligned 512 KB buffers. A 64-byte tail buffer prevents missing signatures that straddle chunk boundaries.

03

Match Magic Number Headers

Each byte is compared against known file headers — e.g. FF D8 FF for JPEG, %PDF for PDF. Office files are detected inside ZIP containers by inspecting internal paths like word/ or xl/.

04

Carve & Save

Once a header is found, bytes are read until a matching footer (e.g. FF D9 for JPEG) or the max size limit is reached. Files are saved in recovered/<type>/ subfolders.

Platforms

Runs everywhere

One codebase — three platforms. The .exe is Windows-only; the Python source runs on macOS and Linux too.

Windows Recommended

Download the standalone .exe — no Python required. Run as Administrator to access raw disks.

macOS

Run from source with sudo python3 app.py. Use an external USB drive — macOS SIP protects the boot volume.

sudo python3 app.py

Linux

Run from source with sudo python3 app.py. Use lsblk to identify your target block device.
